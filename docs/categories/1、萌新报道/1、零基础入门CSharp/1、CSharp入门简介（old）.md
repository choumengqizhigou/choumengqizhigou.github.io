## C#语言介绍

`C#` 作为一门编程语言，首选需要我们提炼出两个关键字，一个 **编程** ，一个 **语言** 。

**编程** 是一种行为， **语言** 是一种方式，一种手段，描述起来就像是我们编写 `C#` 代码来制作一个软件，一个功能，实现跟计算机之间的交流。

这里的语言概念跟我们日常的语言概念有很多相似的地方。

首先语言并非是单一的，生活中除了 `中文` ，还有 `英语` 、 `俄语` 、 `法语` 、 `德语` 。。。

计算机中呢，则有 `汇编语言` 、 `C语言` 、 `C++语言` 、 `C#语言` 、 `Java语言` 、 `Python语言` 。。。

其次就是语法的规则，咱们说话是要遵循一套标准，规则的，不是说随意的拼接一些词语，这样别人也无法理解你要表达什么。遵循语法很重要，在计算机中尤为重要，你说一句 **今天中午饭吃过我了** 虽然有语法错误，可能无伤大雅，别人还是能够明白你的意思，但是在计算机中，你对应语言的语法如果输错了，哪怕一个小小的符号，就会导致你无法与计算机进行交流了！

有相似自然也有不相似，不相似的地方这里只说一个，那就是不管你用什么编程语言，他们最后都是转换成比较统一的格式，`二进制格式` ，因为咱们的计算机只能识别并操作二进制的内容。（ **后面会默认使用这条规则进行说明，了解即可** ）

`日常使用的数学是0-9，一共十个数字，简称为十进制。二进制就可以理解为0-1，只有两个数字。无论你在使用计算机时看到了什么，听到了什么，在计算机内部都只是类似于010010100011这样的二进制序列。至于计算机如何将这个序列转换成图像、声音、游戏，后面专门做文章说明。`


## 编程语言的差异

那么多的编程语言，它们之间有什么异同么？因为看到上面描述，好像都是需要转成 `二进制` 的，为什么还会有多种编程语言呢？

要了解这个问题，其实我们就需要反向考虑下，为什么会有编程语言这个东西，我们知道计算机能够识别的是 `二进制` ，但是 `二进制` 想要表示一些操作，就可能需要一个长的序列，比如 `0010 1011` 表示 `+` 号， `0010 1101` 表示 `-` 号，然后呢 `01` 表示数字 `1` ，`10` 表示数字 `2` 。。。

按照上面的规则，我们想要让计算机帮忙我们计算一个 `1 + 1` ，可能就需要输入一个长序列，如： `01   0010 1011    10` 这样的。

这还仅仅是一个 `1 + 1` 的简单算数运算，我们都需要输入这么多的 `0` 和 `1` ,如果再复杂一点，岂不是要翻天了，那写代码该是多么痛苦的事情啊！

所以呢，这时候编程语言应运而生了，有一个叫做 `汇编语言` 的东西出来了，它现在要这么玩，`add` 就是 `+`, `1` 就是 `1` ， `2` 就是 `2` ，写出来的东西就是 `1 add 1` ， `2 add 2` ，然后呢，这些东西计算机肯定不认识啊，我们已经交代过前提了，这时候呢就需要有个转换的 **东西** ， 来将我写的 `1 add 1` 转换成  `01   0010 1011    10` ，然后再交给计算机执行。

那么这个转换的过程，就叫做 `编译` ，就是将我们写的代码转换成计算机能够识别的机器码，这个帮我们转换的 **东西** 就叫做 `编译器`。

对于上述的这个转变过程，其实也是低效的，之前的描述都是一种抽象,实际应用中要更加的复杂，可以看下下面的示例代码（ **仅作演示，由chatgpt生成** ）

``` x86asm
section .data
    num1 db 1
    num2 db 1
    result db 0

section .text
    global _start

_start:
    mov al, [num1]
    add al, [num2]
    mov [result], al

    ; print result
    mov eax, 4
    mov ebx, 1
    mov ecx, result
    mov edx, 1
    int 0x80

    ; exit program
    mov eax, 1
    xor ebx, ebx
    int 0x80
```

上述代码使用了x86汇编语言，实现了1+1=2，并输出的这样一个例子。这无疑是让人头大的，属实有点太多了，绷不住啊，能不能缩减一点啊？

方案那自然是有的，这里我们举一下 `C语言` ，在 `C语言` 中，我们计算 `1 + 1` 可能只需要一行代码，如： `int a = 1 + 1;` ，其实不仅仅是缩减代码， `C语言` 中还有一些其他便利性操作或者说是思想，咱们可以称之为 **语言特色** ，学习不同的语言，都是为了看这个语言有哪些特色。

现在我们知道代码都需要 `编译` 成本地的机器码，但你不知道是 `编译` 其实是一个比较耗时的过程，有时候哪怕只是修改一点点代码，都需要进行重新 `编译` ，所以这无疑增加了我们的等待时间，随着代码越来越多，我们等待的越来越长。既然这样有没有一种不需要 `编译` ，或者说其他形式的代码呢？

咱们聪明的开发人员总是能想个办法偷个懒，要不我们先写代码，然后运行的时候一行代码或者几行代码进行一次动态的 `编译` 呢？

`Python语言` 就是这其中的一个，我们称之为 **解释型语言** 。这种语言，有一个解释器，运行的时候有解释器来挨个解释代码，将代码解释为机器指令并执行，即运行到哪里，解释到哪里。甚至还可以动态的修改代码！（缺点就是每次运行都需要一行一行解释）

这又会导致一个新的问题，就是程序的性能太差了，一开始全部 `编译` 完成，咱们只管运行就行了，像这样运行一行,解释一行，有点太慢了。

那有没有一种折中的方案呢，弄一个即时编译器，运行的时候去 `编译` ，如果有些代码比较频繁出现，那么我们下次运行的时候，咱们就直接用，不编译了行不行？

总该轮到咱们的 `C#语言` 登场了吧。

其实呢还有一些是为了专门便利某一行业或者某一目的做出来的编程语言，比如数值计算，图形模拟，等等等，

所以你看出来了么，编程语言就是创作者对于问题的理解，是他们想要去改善某种情况针对性开发出的一种工具。 

就跟螺丝刀有一角的 三角的 六角的。。。 有手动的，电动的， 有小马力的，大马力的，所以你能告诉我，编程语言为什么有这么多，为什么不使用同一种编程语言了么？
