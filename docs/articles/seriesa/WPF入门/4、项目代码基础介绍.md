## XAML文件描述

在这个新建的 `WPF` 项目中，我们主要描述的就是 `.xaml` 和 `.xaml.cs` 文件，这两个是配套的，一个是界面描述文件，是以 `.xaml` 结尾的，一个是关联界面的代码文件，是以 `.xaml.cs` 结尾的，可以通过文件前方的 `小三角形` 点击展开，就能看到两个不同的文件。

这两个文件我们都可以分别进行双击打开，目前还不涉及代码的书写，所以 `.xaml.cs` 文件咱们可以暂时忽略，想要看一下界面是什么样子的，直接打开 `.xaml` 文件即可。

不过这里有一点特殊的地方， `.xaml` 的文件是有两个的，一个是 `App.xaml` ，一个是 `MainWindow.xaml` ，双击这两个你看到的界面可能是不一样的。

`App.xaml` 是一个特殊的文件，它是没有界面可以展示的，是我们的资源描述文件，里面可以引入一些当前程序所需要的资源。

![App.xaml截图](/images/e684f23b1e5c253b45f1d01d27c20a09.png)

目前也没有其它什么资源需要我们引入，所以 `App.xaml` 咱们混个眼熟，打开看一下即可。

`MainWindow.xaml` 是我们的界面文件，通过 `VS` 启动展示的那个程序界面就是 `MainWindow.xaml` 这个界面。

双击打开 `MainWindow.xaml` 如下图：

![VS截图](/images/066099640313663026a009b8cf53f558.png)

上方为界面预览窗口，下方为 `XAML` 代码编辑窗口。

**注：在界面操作的部分，我们很容易会误操作，导致界面和文章中的展示界面不一致，这里我们只能尽可能的进行一个修正，不能保证所有的情况都考虑到，所以如果你期间有任何的界面不一致，导致无法进行下去，欢迎你随时来进行交流！**

比如在我们操作 `MainWindow.xaml` 文件的时候，你那边出现了只展示 `预览界面窗口` ，或者只展示 `XAML代码窗口` ，类似于下图这样：

![VS截图](/images/05508d5e0d0257410116ce934ad1d702.png)

这样的话，如果你只有 `预览界面窗口` ，往上拖完之后就会出现 `XAML代码窗口` ，反之亦然。

接下来描述一下我们的界面语言 `XAML` 。

`XAML` 是一种 `类XML` 字符格式，专门用于我们 `WPF` 进行界面描述的。

改变 `MainWindow.xaml` 界面的 `XAML` 内容，就可以同步影响到对应的界面，比如这里进行一些内容的变更，在 `Grid` 标签里面添加了一个 `Button` 标签，并写上 `Hello WPF!` 字样。

``` XML

MainWindow.xaml

<Window ...>
    <Grid>
        <Button
            HorizontalAlignment="Center"
            VerticalAlignment="Center">
            Hello WPF!
        </Button>
    </Grid>
</Window>

```

编辑之后在 `VS` 的 `预览界面窗口` 或者直接 `点击运行` ，都是能看到如下效果的：

![程序运行截图](/images/031f94a0ec19fecb9cd07afa7805acbd.png)

可以看到界面中多了一个上下左右居中的按钮，按钮内部的文字显示 `Hello WPF!` 。


## 代码流程介绍

修改了 `MainWindow.xaml` 文件中的 `XAML` 代码之后，我们感受到这个 `XAML` 确实很神奇，只需要按照特定的格式进行书写，就能得到一些效果，那么后续我们就需要不断的学习这里面的书写方式和所有的控件标签。

目前已知按钮的标签就是 `<Button>` 。

下面我们主要描述的就是界面的启动流程，因为项目的界面文件不会只有 `MainWindow.xaml` 这一个，可能会有多个界面文件，那么启动的时候，哪一个界面文件作为第一个展示的界面，这是是由谁来确定的呢？

答案其实就是 `App.xaml` ，它里面可以给 `Application` 标签写一个属性叫做 `StartupUri` 。

![App.xaml截图](/images/32d6fea0945ac0468937d00a6f5dab99.png)

如果需要其它界面作为初始界面，只需要将 `StartupUri="MainWindow.xaml"` 中的 `MainWindow.xaml` 修改城对应的界面文件名称即可。

为了演示效果，可以创建一个新的界面窗口。

在 `项目WPF_Study右键` -> `添加` -> `窗口` 。

![创建窗口](/images/47b21e4728981412bfb6ed98d07d2080.png)

默认命名 `Windows1.xaml` -> `点击添加` 。

![创建窗口](/images/9f5f8a9d64998aeadad8cd55ce4cce86.png)

此时 `解决方案资源管理器` 中，已经多了一个 `Windows1.xaml` 文件。

![Windows1.xaml 文件图片](/images/18a7cf2dd45054e4270deb09ba4ced8f.png)

这时候可以双击 `MainWindow.xaml` 或者 `Windows1.xaml` 来进行不同窗体界面的编辑了。

为了演示不同的启动窗体，我们做一点内容上的差异，给 `Windows1.xaml` 的 `XAML` 代码也添加一个 `Button` 标签。

``` XML

Windows1.xaml

<Window ...>
    <Grid>
        <Button Content="Hello WPF New Window！" />
    </Grid>
</Window>

```

界面编辑完成之后，剩下的就是去到 `App.xaml` 里面，将 `StartupUri="MainWindow.xaml"` 改为 `StartupUri="Windows1.xaml"` ，然后启动运行项目即可。

具体的运行效果，这里就不贴图了，大家自行感受即可。

另外你会注意到，我们 `XAML` 对于内容的写法可能会有不同的方式，比如第一个 `Button` 的显示内容写在两个标签之间， 第二个 `Button` 是以属性的方式来书写显示内容的，这本质都是一样的，它们改变的都是 `Button` 按钮的 `Content属性` ，只是 `XAML` 为我们提供了一丝丝便利的写法而已。


## 结语

对项目结构和代码流程做一个简单的介绍之后，下面就开始正式进入到 `XAML` 语言的学习中了。

目前你会发现我们忽略了一部分内容的说明，就是 `<Window ...>` 标签中的很多奇怪的属性，比如什么 `x:Class="WPF_Study.MainWindow"` `xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"` ...

这些其实是我们 `XAML` 的命名空间，跟 `C#` 中的 `using` 是一个意思，暂时不在我们的介绍范围内，暂时别动它即可，后续我们会专门来动它的。
