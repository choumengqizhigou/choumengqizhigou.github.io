# 创建项目后的说明

## VS 2022项目流程说明

至此已经创建了你的第一个 `C#` 代码项目，并且能够直接运行，并展示一些内容。

但是这里要做一个说明，就是之前手动 `编译` 代码的时候，其实是生成了一个 `.exe` 结尾的可执行程序的。

我们直接点击 `运行` 是一个什么意思呢？

当你点击 **运行** 的时候，其实 VS 2022 帮你编译了代码，并且生成了 **可执行应用程序** ，并且执行这个 **程序** 。

所以你点击运行的时候，看到的就是这个程序的执行效果。

那这个程序的路径在哪里呢？

如果你没更改过相关的路径设置的话，默认情况下就在我们项目的 `bin` 文件夹目录下。

不过这里可能会有两个文件夹，一个 `Debug` 目录，一个 `Release` 目录，默认是 `Debug` 目录，这些都是可以通过更改 **编译选项** 来控制。

![20230525100829](https://blog-image-group.oss-cn-shanghai.aliyuncs.com/BlogImages/20230525100829.png)

`启动` 旁边就可以快捷的设置这个编译选项。

此时你肯定很好奇 `Debug` 和 `Release` 又有什么区别呢？

`Debug` ：此时的编译程序内部会有一些帮助调试的代码， `VS 2022` 需要这些东西，来给出更多的信息帮忙我们定位问题。

`Release` ：一般我们发布程序，给别人使用就是 **Release** 版本，首先就是没了一些辅助代码，这个体积更小，性能更好。。。

像这样的 **编译选项** 其实还有比较多，但是目前我们了解这两个就可以了，剩下的一些东西，可以等到遇到实际的问题再来尝试修改一些 **编译选项** 。

## 代码结构说明

首先，代码是分为上下两层，上层的 `using` 和下层的 `namespace` ，这些东西目前来说，你都不知道是干什么。

那么我们尝试来做几点说明：

1、 `{}` 是我们代码的范围，目前的范围分为内外三层，这三层依次是（ **using暂且不看** ）

```
namespace FirstCode
│
│  
└─internal class Program
    │
    │  
    └─static void Main(string[] args)
```

仔细观察大括号的范围，是一层一层来的

```
namespace（命名空间）
{
    class（类型）
    {
        Main（方法）
        {

        }
    }
}
```

首先这是一个代码的常规结构， `命名空间` 下有 `类型` ， `类型` 下有 `方法` ，而且是一对多的概念，目前这里只有一对一，不影响。我们后续来增加多个。

2、名称问题， 我们的命名空间叫什么名字，类型叫什么名字，方法叫什么名字，或者说为什么要有这样一个名字的概念呢？那是因为目前我们还没用到，但是后期是需要通过名字来找到它们！

`namespace` `空格` `名称`， `class` `空格` `名称` ，这分别是我们的命名空间名称和类型名称。

所以可以说，当前代码的命名空间是 `FirstCode` ，类型名称是 `Program` ,方法名称稍微有点不一样， 你姑且认为就是 `Main` 。

名称是可以随意修改的，这里你不叫 `FirstCode` `Program` 都可以，随便你怎么改，你可以试试看。

![20230525111527](https://blog-image-group.oss-cn-shanghai.aliyuncs.com/BlogImages/20230525111527.png)

可以叫 `FirstCode666` `Program999` 。

![20230525111646](https://blog-image-group.oss-cn-shanghai.aliyuncs.com/BlogImages/20230525111646.png)

也可以叫 `FirstCodeFirstCode` `ProgramProgram` 。

![20230525111915](https://blog-image-group.oss-cn-shanghai.aliyuncs.com/BlogImages/20230525111915.png)

甚至可以使用中文 `阿猫` `阿狗` 。

也就是说，你想用什么，就用什么，不过有一条准则，就是 `数字` 不能放在第一位

![20230525112039](https://blog-image-group.oss-cn-shanghai.aliyuncs.com/BlogImages/20230525112039.png)

叫 `666FirstCode` `999Program` 就会出现非常恐怖的场面了，全部都是报错。

**所以记住，名字随便叫，但是数字不能打头阵。**


## Main方法

方法呢，其实名字也是可以自定的，随便叫啥都可以，但是这里有一个特例！

就是我们的 `Main` 方法，可以看到两句代码，都是写在这个方法里面的， 这预示着大部分代码的范围，是需要在方法的 `{}` 括号里的。

前面提到，类型中其实可以有多个方法， 这意味着，代码可能也会很多，那么这么多代码，在编译成程序之后，要从哪一行代码执行呢？

这正是创建项目的时候，自动生成的 `Main` 方法，并且 `Main` 方法不能修改名称的原因。

因为 `Main` 方法指示着程序的入口，系统找到 Main 方法，然后执行 `Main` 中的代码！

**这预示着，要想让你的代码起作用，就必须让代码直接或者间接的写在Main方法的范围内。**

这样就能解释，为啥一开始写的两句代码是在 `Main` 的 `{}` 括号范围内了。
