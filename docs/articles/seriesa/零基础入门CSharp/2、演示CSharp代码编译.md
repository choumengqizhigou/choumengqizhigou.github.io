# 演示C#代码编译

## 前言

经过前面的描述，我们了解到写的代码可以经过编译操作变成计算机需要执行的机器码。

不过对于不同的语言来说，它们编译的操作可能有差异，比如咱们的 `C#` ，它并不是直接编译成可以执行的机器码，而是先编译成中间语言 `IL` ，然后再在运行的时候由 `CLR` 中的 `JIT` 来编译成机器码运行的。

这里我们依然采用 `写代码` -> `编译成机器码` -> `程序运行` 的逻辑来进行演示，所以希望你明白这其实是一个包装的过程。

接下来，我将为你演示，如何创建一段 `C#代码` ，并且使用 `C#的编译器` ，来将它转换成一个 `Windows` 平台下的 **可执行应用程序** 。

**此部分仅仅作为演示，看一下即可，不必跟着操作，因为你目前可能还没有C#的编译器。（后面跟着操作安装完开发工具VS2022，可以跟着本节一起动手试试）**

> 可以说一款编程语言的特性、语法几乎都是编译器去决定的，因为它们才是那个翻译官，帮我们跟计算机打交道，它翻译的怎么样，以及支持哪些翻译这至关重要！另外，编译器不是唯一的，很多语言有多款编译器。


## 操作说明

下面操作是按照 `Windows10` 操作系统来的，其主要目的就是为了创建一个 **文本文件** ，并且写上一些代码，保存即可！

> 文本文件的后缀名是 `.txt` ，需要修改文件后缀名为`.cs` ，因为都是文本文件，所以这个后缀名的修改并不影响 `编译` 操作，你不修改也是可以的，只是表明一下这是一个 `C#` 的代码文件。

**文件后缀名的操作在下方。**

不出意外的话，以后看到 `.cs` 结尾的就知道这是一个 `C#` 的代码文件。

当然也没有明确的规定一定是这样的，毕竟谁都可以这么修改文件后缀名。


## 文件扩展名

文件扩展名是用来标识文件类型，协助程序如何处理我们的文件的。

下面演示了部分文件类型的扩展名。

* 文档：`.txt`、`.doc`、`.docx`、`.pdf`
* 图片：`.jpg`、`.jpeg`、`.png`、`.gif`、`.bmp`
* 音频：`.mp3`、`.wav`、`.flac`
* 视频：`.mp4`、`.avi`、`.mkv`
* 压缩：`.zip`、`.rar`、`.7z`
* 程序：`.exe`、`.msi`
* 脚本：`.bat`、`.cmd`、`.ps1`

作为一名开发人员，是很建议开启文件扩展名的，以助于我们后续快速识别文件类型并作出对应操作。

先来看看如何打开文件扩展名的显示。

`双击此电脑` -> `打开资源管理器` -> `点击查看` -> `勾选文件扩展名` 。

![打开文件扩展名](images/8d326f96-62a0-4af1-9edc-9a63e03a8776.png)

**如果不勾选文件扩展名，你是无法进行扩展名的编辑的。**

比如你需要一个 `123.txt` ，未开启文件扩展名的情况，即使手动书写 `.txt` 后缀，那这个后缀也只是作为文件名的一部分。

等真正开启文件扩展名之后，你得到的文件名称可能是 `123.txt.txt` 。

虽然文件扩展名可以标识文件类型，并且我们也能对它进行修改，**但仅仅修改扩展名通常不会更改文件本身的格式** 。

就像你把文本文件 `123.txt` 你修改成 `123.mp4` ，这时候虽然是由视频播放软件来打开该文件，但是播放软件会提示这不是一个正确的视频文件。

接下来为了演示，会把 `.txt` 修改成 `.cs`，这么操作并没有改变文件格式，所以本质上它还是一个文本文件，用记事本依然能以文本文件的格式来打开它，只是为了区别一下，告诉别人这是一个 `C#` 的代码文本文件。

后面安装了开发工具 `VS2022` 之后，你双击 `.cs` 文件，是由 `VS2022` 直接打开这个文本文件的。

## 编译代码

现在代码已经写好了，我们只需要调用 `编译器` ， `编译` 一下代码就行了，下面我将演示 `编译器` 的使用。

**没有安装 Visual Studio 2022，这里可能没有编译器软件。**

默认的编译器路径在 `C:\Program Files\Microsoft Visual Studio\2022\Professional\Msbuild\Current\Bin\Roslyn` 这个文件夹下，有一个 `csc.exe` 这个程序， 这就是我们的 `C#编译器` 

> .exe结尾的文件是 Windows 下的可执行应用程序，一般我们双击该文件就能运行程序了。
> 不过有些程序没有独立的界面，可能是以服务的形式运行的，你双击之后什么都看不到，还有一些程序是需要一些启动参数，没有配置启动参数的话，双击之后程序会立即关闭，也什么都不到。

![编译器软件路径截图](images/a302075e-f1f4-4b18-9ac3-14af80bcb079.png)

有了 `编译器` 就可以开始 `编译` 我们的代码了！

这里有一个问题就是，这个 `编译器` 程序并不能直接双击运行，需要一些启动参数，可以在 `cmd` 窗口中配置启动参数去运行它。

## CMD窗口运行csc.exe

直接在地址栏输入 `cmd` ，就会启动 `cmd` 命令窗口，并且自动定位到当前目录。

![地址栏输入cmd](images/382ab69f-ba21-4e03-a329-9ee8a1a3f1bc.png)

![cmd命令演示](images/281b5396-05e3-4844-803b-0b5e2124fcbe.png)

目前这个命令行已经定位到我刚才打开的文件夹地址了，下面直接输入对应要启动的程序名称和附带的启动参数即可。

启动程序和附带参数的格式如下：

``` cmd
[程序名称] [参数1] [参数2]...
```

**程序名称和参数之间用 空格 隔开。**

因为我们只做默认编译的参数设置，所以只需要提供两个参数即可，一个是需要编译的文件路径，一个是输出编译之后的文件路径。

需要编译的代码文件名是我们自己命名的，叫 `FirstCsharpCode.cs`，并且是在桌面创建的，所以文件的完整路径就是 `C:\Users\LIFEI\Desktop\FirstCsharpCode.cs` 。

**每个人的桌面路径是不一样的，请确认你那边的桌面路径。**

然后是我们 `编译` 后的文件输出路径，我们需要把这段代码编译成一个 **可执行的应用程序** ，并且输出到桌面上，所以路径就直接是桌面路径再加上输出的文件名称。

输出的文件名称跟之前创建的文件名称一样，随便我们怎么叫，我们这里就取名为 `FirstCsharpCode.exe` ，不过输出需要加一个标识符号 `/out:` ，那么完整的输出文件路径就是 `/out:C:\Users\LIFEI\Desktop\FirstCsharpCode.exe` 。

**输入的是.cs，输出的是.exe。**

现在三个要素都有了：

* 启动程序：`csc.exe` （可以只写csc）
* 需要编译的文件路径：`C:\Users\LIFEI\Desktop\FirstCsharpCode.cs`
* 编译后的文件路径：`/out:C:\Users\LIFEI\Desktop\FirstCsharpCode.exe`

剩下的就是组合这几个参数，输入到 `cmd` 里面，然后敲击回车执行就可以了。

```
csc C:\Users\LIFEI\Desktop\FirstCsharpCode.cs /out:C:\Users\LIFEI\Desktop\FirstCsharpCode.exe 
```

**三段内容之间一定要用 空格 间隔。**

回车之后，如果代码编译有错误，会直接在 `cmd` 中提示具体的错误信息。

没有错误就像下图这样，代表编译成功了。

![编译代码成功](images/500aff7b-a9dc-44ba-9a3a-97bc3fea3bee.png)

此时你会看到桌面多出来一个文件，就是我们 `编译` 后的输出文件 `FirstCsharpCode.exe` 。

![编译生成文件截图](images/e1d8e879-b5ce-41c9-b91f-0d1624a330ed.png)

咱们双击运行这个程序，可以看到程序弹出一个黑框，里面输出了 `这是我的第一个程序` 这样的文字！

![编译生成文件截图](images/6e7f8358-31bf-4d60-ae08-3024418c2aec.png)

敲击回车，或者点击右上角的 `X` 就可以关闭这个程序了。

> 很多人会纠结，为啥输出的信息不是Hello World!
> 哈哈哈，这是我的错，你也可以调整输出的内容，但是千万不要乱修改其它部分，输出的内容一定要放在""之间。
