## C#语言介绍

`C#` 作为一门编程语言，首选需要我们提炼出两个关键字，一个 **编程** ，一个 **语言** 。

**编程** 是一种行为， **语言** 是一种方式，一种手段，描述起来就像是我们编写 `C#` 代码来制作一个软件，一个功能，实现跟计算机之间的交流。

这里的语言概念跟我们日常的语言概念有很多相似的地方。

首先 **语言** 并非是单一的，生活中除了 `中文` ，还有 `英语` 、 `俄语` 、 `法语` 、 `德语` ...

计算机中呢，则有 `汇编语言` 、 `C语言` 、 `C++语言` 、 `C#语言` 、 `Java语言` 、 `Python语言` ...

其次就是语法的规则，咱们说话是要遵循一套标准、规则的，不是说随意的拼接一些词语，不然可能导致别人无法准确领会你的意思。

**遵循语法很重要，在计算机中尤为重要。**

日常用语中，你说一句 **今天中午饭吃过我了** 虽然有语法错误，别人还是能够明白你的意思。但是在计算机中，你对应语言的语法如果输错了，哪怕一个小小的符号，就会导致你无法与计算机进行交流了！

两种语言既有相似，自然也有不相似。

相似的地方这里只说一个，那就是不管你用什么编程语言，它们最后都会转换成比较统一的格式，`二进制格式` ，因为咱们的计算机只能识别并操作 `二进制` 的内容。

`小知识点：日常使用的数学是0-9，一共十个数字，简称为十进制。二进制就可以理解为0-1，只有两个数字。无论你在使用计算机时看到了什么，听到了什么，在计算机内部处理的都只是类似于010010100011这样的二进制序列。至于计算机如何将这个序列转换成图像、声音、游戏，后面咱们专门来聊。`


## 从二进制开始

从上面的描述来看，首先编程语言好像有很多种，其次不管你用什么编程语言，好像都是需要转成计算机能够处理的 `二进制` 内容。

既然这样，很直观的一个疑问立马就变成了：

**为什么需要这么多的编程语言呢？**

这个疑问其实不难解释，只不过需要咱们换个思路方向，反向考虑一下，为什么会有编程语言这个东西。

我们知道计算机能够识别的是 `二进制` ，但是 `二进制` 想要表示一些操作，就可能需要一个长的序列。

这里我们举一个案例缩影，来大概描述一下这种概念。比如 `0010 1011` 表示 `+` 号， `0010 1101` 表示 `-` 号，然后呢 `01` 表示数字 `1` ，`10` 表示数字 `2` 。。。

按照上面的规则，我们想要让计算机帮忙我们计算一个 `1 + 1` ，可能就需要输入一个长序列，如： `01   0010 1011  10` 这样的。

这还仅仅是一个 `1 + 1` 的简单算数运算，都需要输入这么多的 `0` 和 `1` ,如果再复杂一点，岂不是要翻天了，那写代码该是多么痛苦的事情啊！

所以呢，这时候编程语言应运而生了，有一个叫做 `汇编语言` 的东西出来了。

汇编的处理逻辑很简单，要用一些人们更容易理解的概念来书写 `1 + 1`。

比如在汇编中，`add` 就是 `+`, `1` 就是数字 `1` ， `2` 就是数字 `2` ，写出来的东西就是 `1 add 1` ， `2 add 2` 。

这样势必会导致一个问题，就是你写的这些内容，计算机压根看不懂。

因为前面咱们已经交代过了，计算机只认识 `二进制` ，所以这时候呢，就需要有个转换的 **东西** ， 来将你书写的 `1 add 1` 转换成  `01   0010 1011    10` 这样的序列，再交给计算机来执行。

那么这个转换的过程，就叫做 `编译` ，就是将我们写的代码转换成计算机能够识别的机器码，这个帮我们转换的 **东西** 就叫做 `编译器`。

即便引入了汇编语言，对于咱们写代码来说，也是极其低效的。

仅仅实现一些简单的功能，可能还勉强能写能看，要是涉及到一些应用开发，就需要咱们引入更多的抽象，来让越少的代码，能够做成更大的事情。 

这里先看一段汇编代码。（ **仅作演示，由chatgpt生成** ）

``` asm
section .data
    num1 db 1
    num2 db 1
    result db 0

section .text
    global _start

_start:
    mov al, [num1]
    add al, [num2]
    mov [result], al

    ; print result
    mov eax, 4
    mov ebx, 1
    mov ecx, result
    mov edx, 1
    int 0x80

    ; exit program
    mov eax, 1
    xor ebx, ebx
    int 0x80
```

上述代码使用了x86汇编语言，实现了1+1=2，并输出的这样一个例子。这无疑是让人头大的，属实有点太多了，绷不住啊，能不能缩减一点啊？

下面我们来请下一位选手登场，这里我们推举一下 `C语言` ，在 `C语言` 中，我们计算 `1 + 1` 可能只需要一行代码，如： `int a = 1 + 1;` 。

无需多言，这种爽快的感觉是不是让人直接起飞！

对于咱们的 `C语言` 来说，这里不仅仅是缩减代码，语言本身还会有一些其他便利性操作或者说是设计思想，咱们可以称之为 **语言特色** ，学习不同的语言，其实主要是为了看这个语言有哪些特色。


## 语言的差异

现在我们知道代码都需要 `编译` 成本地的机器码，但你不知道是 `编译` 其实是一个比较耗时的过程，有时候哪怕只是修改一点点代码，都需要进行重新 `编译` ，所以这无疑增加了我们的等待时间，随着代码越来越多，我们等待的越来越长。既然这样有没有一种不需要 `编译` ，或者说不需要立即 `编译` 的方式呢？

聪明的开发人员总是能想个办法偷个懒，要不我们先写代码，然后运行的时候一行代码或者几行代码进行一次动态的 `编译` 呢？

`Python语言` 就是这其中的一个，我们称之为 **解释型语言** 。这种语言，有一个解释器，运行的时候有解释器来挨个解释代码，将代码解释为机器指令并执行，即运行到哪里，解释到哪里。甚至还可以动态的修改代码！（缺点就是每次运行都需要一行一行解释）

这又会导致一个新的问题，就是程序的性能太差了，一开始全部 `编译` 完成，咱们只管运行就行了，像这样运行一行,解释一行，有点太慢了。

那有没有一种折中的方案呢，弄一个即时编译器，运行的时候去 `编译` ，如果有些代码比较频繁出现，那么我们下次运行的时候，咱们就直接用，不编译了行不行？

总该轮到咱们的 `C#语言` 登场了吧。

其实呢还有一些是为了专门便利某一行业或者某一目的做出来的编程语言，比如数值计算，图形模拟，等等等...

所以你看出来了么，编程语言就是创作者对于问题的理解，是他们想要去改善某种情况针对性开发出的一种工具。 

就跟螺丝刀有一角的 三角的 六角的。。。 有手动的，电动的， 有小马力的，大马力的。
